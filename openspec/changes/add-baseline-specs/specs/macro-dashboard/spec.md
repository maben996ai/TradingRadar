## ADDED Requirements

### Requirement: 宏观指标看板
系统 SHALL 提供 `GET /api/macro/indicators` 接口，返回宏观指标看板：对每个注册指标给出最新值、前值、变化、判断结论、一句话理由、所处区间（zone）及完整历史序列。

#### Scenario: 获取看板
- **WHEN** 已登录用户调用看板接口
- **THEN** 系统基于已入库的观测数据，对每个注册指标计算最新值/前值/变化/判断/区间并附历史序列返回

#### Scenario: 指标暂无数据
- **WHEN** 某指标尚无任何观测记录
- **THEN** 系统对该指标返回空值或缺省，不影响其他指标返回

### Requirement: 宏观指标注册表与判断引擎
系统 SHALL 以代码内注册表维护宏观指标元信息（指标键、方向、预警阈值、说明等），并以确定性规则引擎对最新值给出判断结论与所处区间（高位/正常/低位）。指标元信息不入库。

#### Scenario: 规则判断
- **WHEN** 计算某指标的判断与区间
- **THEN** 系统按注册表中的阈值规则输出确定性结论（不依赖大模型）

### Requirement: 宏观观测数据刷新
系统 SHALL 提供 `POST /api/macro/refresh` 接口，从外部数据源（FRED）拉取各指标观测并以 (indicator_key, date) 为唯一键 upsert 入库，返回新增条数。

#### Scenario: 成功刷新
- **WHEN** 已配置数据源凭证的情况下调用刷新接口
- **THEN** 系统拉取观测、按唯一键去重 upsert 并返回新增条数

#### Scenario: 数据源不可用或未配置凭证
- **WHEN** 外部数据源调用失败或未配置 API key
- **THEN** 系统返回 502 并附错误信息，不写入部分脏数据

### Requirement: 宏观数据定时与首启刷新
系统 SHALL 通过调度器每日刷新宏观观测，并在数据表为空且已配置凭证时于服务首启时自动拉取一次。

#### Scenario: 每日定时刷新
- **WHEN** 每日调度时点到达
- **THEN** 系统执行宏观观测刷新

#### Scenario: 首启空表自动拉取
- **WHEN** 服务启动时观测表为空且已配置凭证
- **THEN** 系统自动执行一次刷新；未配置凭证时跳过，不触发外部调用
